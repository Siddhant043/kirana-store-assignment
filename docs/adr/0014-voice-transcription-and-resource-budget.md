# Voice via hosted Whisper; resource budget for a 2 vCPU / 4 GB VPS

The production VPS is 2 vCPU / 4 GB RAM / 100 GB SSD, which must run Postgres, the async agent process, WeasyPrint (Pango/Cairo), and python-pptx. Voice-note transcription is **offloaded to a hosted Whisper API** (OpenAI/Groq) rather than a resident local model — this keeps memory predictable and leaves the two cores for the agent loop and CPU-bound document rendering. A local `faster-whisper` remains a viable single-vendor fallback on the 4 GB box (lazy-loaded, unloaded after use), but the hosted API is the default so a voice note never contends for RAM/CPU with a concurrent PDF or deck render. Voice is a stretch feature and must not threaten core stability; transcription output feeds the normal agent loop as if the owner had typed it, so no core logic changes. This accepts a second vendor and API key as a deliberate trade for predictable memory. Related resource posture on this box: Postgres is tuned small (`shared_buffers ~128MB`), and heavy document generation (deck + invoice) is serialized so two large renders can't spike RAM simultaneously. The self-contained fallback (lazy-loaded `faster-whisper tiny/base` int8, unloaded after use) is documented but not the default.
