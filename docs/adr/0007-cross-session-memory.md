# Cross-session memory in Postgres, keyed by Telegram user id, injected each turn

Standing preferences (default payment, preferred brand) and the shop profile (name, GSTIN) live in Postgres (`preferences`, `shop_profile`) keyed by the owner's Telegram user id — outside the context window, so a `/new` chat clears the conversation but not what the store knows. Two access paths: the current preferences are rendered into the system prompt each turn (defaults apply with no tool call), and `set_preference` / `get_preferences` tools make writes the source of truth (captured by a tool, not a regex). An explicit in-message instruction (". . . cash") overrides the stored default for that one bill without changing it. Keyed single-owner-per-chat deliberately; multi-tenant-per-shop is out of scope.
